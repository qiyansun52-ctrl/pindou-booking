设计 — 第 1 节：数据模型

Supabase PostgreSQL 数据库表设计

表 1：available_slots（可预约时间段）
字段 类型 说明
id UUID 主键
date DATE 日期，如 2026-04-15
start_time TIME 开放开始时间，如 10:00
end_time TIME 开放结束时间，如 20:00
is_active BOOLEAN 是否启用

店家设置某天的营业时间。比如 4 月 15 日 10:00-20:00 开放预约。顾客只能在这个范围内选择开始时间。

表 2：bookings（预约记录）
字段 类型 说明
id UUID 主键
slot_id UUID → available_slots 关联哪天的开放时间
customer_name TEXT 顾客姓名
contact_type ENUM 'wechat' 或 'whatsapp'
contact_value TEXT 微信号或 WhatsApp 号码
start_time TIMESTAMPTZ 预约开始时间
duration_hours INTEGER 预约时长（小时）
num_people INTEGER 人数（1-4）
total_price DECIMAL 自动计算的总价（马币）
status ENUM pending / confirmed / rejected / cancelled
created_at TIMESTAMPTZ 创建时间
表 3：admin_users（管理员）
字段 类型 说明
id UUID 关联 Supabase Auth 用户 ID
email TEXT 登录邮箱
name TEXT 管理员名称

使用 Supabase Auth 进行登录认证，只有 admin_users 表中的用户才能访问后台。

冲突检测逻辑

当顾客提交预约时，API 在服务端执行以下检查：

  1. 计算预约的时间范围：start_time 到 start_time + duration_hours
  2. 查询该时间范围内所有 pending + confirmed 状态的预约
  3. 按每个小时统计已占座位数
  4. 如果任何小时内 已占座位 + 本次人数 > 4，则拒绝预约
