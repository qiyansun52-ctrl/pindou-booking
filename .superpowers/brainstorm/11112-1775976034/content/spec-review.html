概述

为拼豆岛（PINSLAND）构建一套线上预约系统，包含顾客端 H5 网页和管理后台。顾客通过 WhatsApp 分享的链接进入预约页面，选择时间和人数后提交预约，店家在后台审核确认。

业务背景

品牌：拼豆岛 PINSLAND（SHERVIE ART）
地址：JAYAONE | UM，Pacific Star
微信：PINSLAND
座位数：4 个
收费：线下结算（1h 19.9 RM / 2h 36.6 RM / 3h 49.9 RM，开业特惠首小时 9.9 RM）
系统不处理价格计算，仅负责预约和座位管理，收入由店家手动录入

技术选型

框架Next.js 14 (App Router) UITailwind CSS + shadcn/ui 数据库Supabase (PostgreSQL) 认证Supabase Auth 实时Supabase Realtime 部署Vercel（免费） 语言TypeScript

数据模型

表 1：available_slots（可预约时间段）
idUUID主键
dateDATE日期
start_timeTIME开放开始时间
end_timeTIME开放结束时间
is_activeBOOLEAN是否启用
表 2：bookings（预约记录）
idUUID主键
slot_idUUID关联 available_slots
customer_nameTEXT顾客姓名
contact_typeENUM'wechat' 或 'whatsapp'
contact_valueTEXT微信号或 WA 号码
start_timeTIMESTAMPTZ预约开始时间
duration_hoursINTEGER预约时长（小时，不限上限）
num_peopleINTEGER人数（1-4）
actual_amountDECIMAL实际收款（店家手动录入）
statusENUMpending / confirmed / rejected / cancelled
created_atTIMESTAMPTZ创建时间
表 3：admin_users（管理员）
idUUID关联 Supabase Auth
emailTEXT登录邮箱
nameTEXT管理员名称

冲突检测逻辑

  1. 计算预约时间范围：start_time 到 start_time + duration_hours
  2. 查询该范围内所有 pending + confirmed 状态的预约
  3. 按每个小时统计已占座位数
  4. 如果任何小时内「已占座位 + 本次人数 > 4」，拒绝并提示座位不足

页面结构

顾客端（H5，无需登录）
1. 首页 /
店铺信息 + 日历选日期（绿/灰/不可点）
2. 预约表单 /book?date=
座位时间轴 → 选时间/时长/人数 → 填姓名 + 联系方式（微信或WA）→ 提交
3. 预约状态 /booking/[id]
预约详情 + 实时状态更新（待确认/已确认/已拒绝）
管理后台（需登录，移动端优先）
预约列表 /admin
查看/确认/拒绝/取消 + 录入金额 + 实时通知
时间管理 /admin/slots
设置开放时间 + 批量设置一周 + 启用/停用
数据统计 /admin/stats
日/周/月人数 + 收入统计 + 趋势图表
登录 /admin/login
邮箱 + 密码，Supabase Auth

预约流程

1
店家后台设置可预约日期和时间范围
2
顾客打开 H5，日历显示实时可用日期
3
选日期 → 看时间轴 → 选时间/时长/人数 → 填信息 → 提交
4
系统冲突检测，通过后创建 pending 预约
5
店家收到通知，确认或拒绝
6
顾客实时看到预约状态
7
到店线下结算，店家录入收款金额
